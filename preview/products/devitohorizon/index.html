DevitoHorizon · Python FWI & imaging SDK

A complete FWI & imaging SDK — to configure, not build.

DevitoHorizon is a collection of composable building blocks for seismic FWI and imaging that together form a complete inversion SDK — data handling, misfits, optimizers, preprocessing, gradient conditioning and multi-scale orchestration. It delegates the wave-equation solves to DevitoPRO, so a geophysics team can assemble a full FWI/RTM/LS-RTM workflow from tested, composable components.

What DevitoHorizon adds

The full workflow around the wave equation

The same workflow runs as a composable Python API (Dataset, DevitoPhysics, Optimizer) or a command-line driver.

Misfits

Least-squares (L2), L1, Student’s-t, phase and envelope objectives, with normalization wrappers and a hook for user-supplied misfit/gradient pairs.

Optimizers

Spectral Projected Gradient, Projected Quasi-Newton (L-BFGS), a split-Bregman sparsity scheme and SciPy wrappers — all box-constrained.

Physics

Iso-acoustic, isotropic-elastic and TTI (elastic & acoustic) propagators, selected automatically from the model — plus a hook for user-supplied physics.

Workflow

Frequency continuation for multi-scale inversion, stochastic mini-batching over shots, and multi-parameter inversion with per-parameter bounds.

Conditioning & preprocessing

Water-bottom / datum / cone muting, illumination preconditioning, source-wavelet estimation, band-pass, AGC and normalization — applied consistently to observed and synthetic data.

Data & models

SEG-Y and Zarr shot data, HDF5/SEG-Y/Zarr models, streamed per shot for predictable memory use.

Design philosophy

Explicit, composable, swappable

Components are wired together as first-class objects — no hidden state, no monolithic driver. Every major piece (misfit, optimizer, scheduler, data backend, gradient processor) is swappable through a small interface, so you can drop in a proprietary misfit, a custom I/O backend or an in-house optimizer without forking the core. The reference algorithms are short and readable — bridging method development and production.

Kernel performance and hardware portability belong to DevitoPRO: DevitoHorizon generates no device code. Its parallelism is shot-level task scheduling — serial by default, or distributed with Dask — which composes with, and is distinct from, DevitoPRO's per-device acceleration.
